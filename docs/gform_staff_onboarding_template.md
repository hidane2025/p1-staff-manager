# P1 Staff Onboarding — Google フォーム 推奨質問テンプレート

P1事業の受付運用で使う「スタッフ事前情報フォーム」の推奨質問セット。
Google フォーム に以下の質問をそのままコピペすれば、出力 CSV を P1 Staff Manager の
「🔗 Googleフォーム連携」タブからそのまま取り込めるように設計されている。

- 取り込みスクリプト: `utils/gform_importer.py`
- 取り込み画面: `pages/1_スタッフ管理.py` → 「📥 スタッフ一括登録/更新」→「🔗 Googleフォーム連携」タブ
- Google フォームの既定 CSV 書式（タイムスタンプ列＋各質問を列名とする書式）に対応

---

## 運用フロー

1. Google フォームにこのテンプレの質問をコピペして作成
2. フォームの URL を業務委託契約者/スタッフに配布（LINE / メール）
3. 回答が集計される Google スプレッドシート を「ファイル → ダウンロード → CSV」で保存
4. P1 Staff Manager「📋 スタッフ管理」ページで「🔗 Googleフォーム連携」タブを開き、CSV をアップロード
5. プレビューで内容を確認してから「🔄 P1にインポート」ボタンで登録

マイナンバー等の機微情報は **取得しない**（プライバシー配慮）。

---

## 質問一覧（Google フォームにコピペ）

> 末尾に `→ <内部カラム>` と付いているのは P1 Staff Manager 側のマッピング先カラム。
> フォーム作成時はこのコメントは削除してよい（スクリプト側は質問文で照合する）。

### 1. お名前（本名・漢字） *必須*
- 形式: 記述式（短文）
- ヘルプ: 領収書・契約書に記載される氏名を入力してください
- → `real_name`

### 2. お名前（カタカナ） *必須*
- 形式: 記述式（短文）
- ヘルプ: 振込先照合用。全角カタカナで入力してください
- → `notes`（フリガナ欄として `notes` に統合）

### 3. ディーラーネーム（現場での呼び名）
- 形式: 記述式（短文）
- ヘルプ: 大会中に使うリングネーム。未入力の場合は本名を使います
- → `name_jp`

### 4. 生年月日 *必須*
- 形式: 日付
- → `notes`

### 5. 性別
- 形式: ラジオボタン
- 選択肢: 男性 / 女性 / その他 / 回答しない
- → `notes`

### 6. メールアドレス *必須*
- 形式: メールアドレス
- ヘルプ: 領収書送付に使います
- → `email`

### 7. 電話番号 *必須*
- 形式: 記述式（短文）
- ヘルプ: ハイフンありなしどちらでも可
- → `contact`

### 8. LINE ID または LINE QR リンク
- 形式: 記述式（短文）
- ヘルプ: 緊急連絡・シフト調整に使います
- → `contact`（電話番号と統合、電話優先）

### 9. 緊急連絡先（氏名・電話番号）
- 形式: 記述式（段落）
- ヘルプ: 本人と連絡が取れない場合の連絡先
- → `notes`

### 10. 郵便番号
- 形式: 記述式（短文）
- ヘルプ: 例: 100-0001
- → `address`（住所と結合）

### 11. ご住所 *必須*
- 形式: 記述式（段落）
- ヘルプ: 都道府県から番地・建物名まで記入してください（交通費計算に使います）
- → `address`

### 12. 最寄駅
- 形式: 記述式（短文）
- ヘルプ: 例: 名古屋駅
- → `nearest_station`

### 13. 役職（複数選択可）
- 形式: チェックボックス（複数選択）
- 選択肢: ディーラー / フロア / TD / チップ / DC
- ヘルプ: 対応可能なポジションをすべて選択してください
- → `role`（最初の1つ） + `notes`（2つ目以降）

### 14. 雇用区分 *必須*
- 形式: ラジオボタン
- 選択肢: 業務委託 / タイミー / 正社員
- → `employment_type`（内部コード: contractor / timee / fulltime）

### 15. タイミーの場合の希望時給
- 形式: 記述式（短文）
- ヘルプ: 数字のみ（例: 1500）。タイミー以外は空欄
- → `custom_hourly_rate`

### 16. MIX対応可否
- 形式: ラジオボタン
- 選択肢: 可 / 不可 / 条件付きで可
- → `notes`

### 17. 稼働可能曜日（複数選択可）
- 形式: チェックボックス（複数選択）
- 選択肢: 月 / 火 / 水 / 木 / 金 / 土 / 日 / 祝日
- → `notes`

### 18. 開始希望日
- 形式: 日付
- ヘルプ: 最短で稼働可能な日を選んでください
- → `notes`

### 19. 過去の大会運営経験
- 形式: 記述式（段落）
- ヘルプ: 経験があれば大会名・役職・時期を記入してください
- → `notes`

### 20. その他（質問・希望等）
- 形式: 記述式（段落）
- ヘルプ: 何か気になることがあれば自由に記入してください
- → `notes`

## 既知の制約

- フォームの質問を **書き換える場合**、`utils/gform_importer.py` の `_QUESTION_MAP` に
  新しい質問文を追記する必要がある。元の質問文と似た別名を許容するため部分一致で照合している。
- `bulk_import_staff` は NO. かディーラーネーム（`name_jp`）で既存スタッフを照合する。
  同一人物が二重送信した場合は、同じ `name_jp` で **上書き更新**される動作になる。
- マイナンバー、本籍、銀行口座等の機微情報は **取得しない**。必要な場合は別途
  紙面・対面で取得し、安全な経路で保管すること。
