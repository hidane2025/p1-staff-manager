## 1. 概要

P1 Staff Managerは、ポーカー大会のスタッフ（ディーラー・フロア・チップ等）の給与計算・出退勤管理・封筒準備・精算レポートを一元管理するツールです。

**解決する課題:**
- シフト表から1人ずつ手計算していた給与計算を自動化
- 最終日の支払い作業を5分/人 → 30秒/人に短縮
- 銀行で準備する紙幣の枚数を事前に把握
- 欠勤・遅刻・早退・凍結退勤の記録と給与への自動反映
- 承認フローによる不正支払いの防止

**対象ユーザー:**

| 役割 | 主な操作 |
|------|----------|
| 経理（小島・伊藤） | シフト取込・支払い計算・封筒準備・精算レポート |
| 現場責任者（TD） | 出退勤管理・凍結退勤・当日スタッフ追加 |
| 承認者（半谷・中野） | 支払い承認 |

## 2. 動作環境・起動方法

### 必要環境

- Python 3.9以上
- ブラウザ（Chrome推奨）

### 起動手順

```bash
cd P1事業/p1-staff-manager
python3 -m streamlit run app.py --server.port 8502
```

ブラウザで `http://localhost:8502` を開く。

同一ネットワーク内の他端末（タブレット等）からは `http://[PCのIPアドレス]:8502` でアクセス可能。

### 停止方法

ターミナルで `Ctrl + C` を押す。

### データの場所

- データベース: `p1-staff-manager/p1_staff.db`（SQLiteファイル1つ）
- バックアップ: `cp p1_staff.db p1_staff_backup_$(date +%Y%m%d).db`

## 4. イベント準備（大会前）

### 4-1. イベントを作成する

1. 「📅 シフト取込」ページを開く
2. 「新規イベント作成」タブを選択
3. 以下を入力して「作成」を押す

| 項目 | 入力例 | 説明 |
|------|--------|------|
| イベント名 | P1 Nagoya 2026 年末 | 大会名 |
| 会場 | 中日ホール（名古屋） | 会場名 |
| 開始日 | 2025-12-29 | 初日 |
| 終了日 | 2026-01-04 | 最終日 |

### 4-2. 日別レートを設定する

同じ「📅 シフト取込」ページの「日別レート設定」セクションで、各日のレートを設定する。

| 項目 | 通常日の例 | 特別日の例 | 説明 |
|------|-----------|-----------|------|
| 日付 | 2025-12-29 | 2025-12-31 | YYYY-MM-DD形式 |
| 通常時給 | 1500 | 1600 | 22時前の時給 |
| 深夜時給 | 1875 | 2000 | 22時以降の時給 |
| 交通費 | 1000 | 1000 | 1日あたり定額 |
| フロア手当 | 3000 | 5000 | Floor役職のみ |
| MIX手当 | 1500 | 1500 | MIXテーブル担当日のみ |
| 日の種類 | regular | premium | 管理用ラベル |

**注意:** シフト取込時にレート未設定の日は自動でデフォルト値（時給¥1,500）が設定される。特別日は手動で上書きすること。

## 5. シフト取込

### 5-1. CSVの準備

Google Sheetsのシフト表を「ファイル → ダウンロード → カンマ区切り(.csv)」でダウンロードする。

**必要なCSVの列構成:**

| 列 | 内容 | 例 |
|----|------|-----|
| A列 | 役職 | TD / Floor / Dealer / Chip / DC |
| C列 | スタッフNO. | 18 |
| D列 | 名前（日本語） | EveKat |
| E列 | 名前（英語） | EVEKAT |
| F列以降 | 日付ごとの勤務時間 | 13:00~23:00 |

- 休みの日は「×」と入力
- 時刻は「13:00~23:00」「13:00〜23:00」「13:00-23:00」のいずれの形式にも対応
- 深夜は「25:00」「26:00」のように24時超で表記可能

### 5-2. 取り込み手順

1. 「📅 シフト取込」ページを開く
2. イベントを選択
3. 「年」を入力（12月の年を入力。例: 12/29〜1/4なら「2025」）
4. CSVファイルをアップロード
5. プレビューで内容を確認
6. 「🚀 取り込み実行」を押す

**取り込まれるデータ:**
- スタッフ（未登録なら自動作成）
- 各日のシフト（開始・終了時刻）
- レート未設定日は自動設定

### 5-3. 取り込み済みデータの確認

ページ下部「取り込み済みシフト」セクションで全シフトを一覧確認できる。

## 6. 支払い計算

### 6-1. 計算を実行する

1. 「💰 支払い計算」ページを開く
2. イベントを選択
3. 「🔄 支払い額を計算」ボタンを押す

**計算される項目:**

| 項目 | 計算方法 |
|------|----------|
| 基本給 | 通常時間（22時前）× 通常時給 |
| 深夜手当 | 深夜時間（22時以降）× 深夜時給 |
| 休憩控除 | 6h超: -45分 / 8h超: -60分（通常時間から控除） |
| 交通費 | 出勤日数 × 交通費日額 |
| フロア手当 | Floor役職の出勤日数 × フロア手当日額 |
| MIX手当 | MIXフラグがONの日数 × MIX手当日額 |
| 精勤手当 | 全日出勤: ¥10,000 / 4日以上: ¥6,000 |
| **合計** | **上記すべての合計** |

### 6-2. 計算結果の確認

- **サマリー:** 総支払額・対象人数・平均・未承認/承認済/支払済の人数
- **内訳合計:** 基本給・深夜・交通費・フロア・MIX・精勤の各合計
- **一覧テーブル:** 全スタッフの金額一覧（名前・役職でフィルタ可能）
- **個別詳細:** スタッフを選択して内訳を確認

### 6-3. 再計算について

- 出退勤で遅刻・早退・欠勤を記録した後、**必ず再計算ボタンを押す**
- **承認済み・支払済みのスタッフは再計算でスキップされる**（上書き保護）
- 再計算が必要な場合は、先に承認を取り消してから再計算すること

## 7. 出退勤管理（大会中）

### 7-1. 基本の流れ（1日5分で完了）

```
① 「全員出勤」ボタンを押す（現在時刻以前の予定者のみ対象）
② 来てない人だけ「欠勤」タブで×にする
③ 遅刻・延長・早退・凍結があれば該当タブで記録
④ 最終日：「全員退勤」で予定時刻一括確定
```

**原則：シフト通り＝デフォルト。例外だけ記録する。**

### 7-2. 一括操作

| ボタン | 動作 | 注意 |
|--------|------|------|
| ✅ 現在時刻までの予定者を出勤 | 現在時刻以前にplanned_startがあるスタッフのみチェックイン | 未来のシフトはスキップ |
| 🔴 全員退勤（予定時刻で確定） | 全スタッフをplanned_endで退勤確定 | 延長者は事前に記録しておくこと |
| 🔄 リセット | 全員を未確定に戻す | **確認ダイアログあり。慎重に** |

### 7-3. 例外の記録

**❌ 欠勤タブ:**
- 来なかった人を複数選択して「欠勤にする」
- 欠勤者は支払い計算から除外される
- 精勤手当にも影響する

**⏰ 遅刻タブ:**
- スタッフを選択 → 実際の到着時刻（時/分）を入力
- 29時まで対応（深夜勤務対応）
- 遅刻分の給与は再計算時に自動減額

**⏩ 延長タブ:**
- 予定より遅くまで働いた人の実際の退勤時刻を入力
- 延長分の給与は再計算時に自動加算（深夜割増も自動適用）

**⏪ 早退タブ:**
- 予定より早く帰った人の実際の退勤時刻を入力
- 早退分の給与は再計算時に自動減額

**🧊 凍結退勤タブ:**
- トーナメントのテーブルが閉じた時に使用
- 複数スタッフを一括選択 → 同一時刻で一括退勤
- 例: テーブル凍結で10名を20:00で一括退勤

### 7-4. 当日スタッフ追加

予定外のスタッフが急遽参加した場合に使用。

1. 「②-B 当日スタッフ追加」セクションへ
2. 「既存スタッフから選択」or「新規スタッフを作成」を選択
3. 開始・終了時刻を入力（時/分、29時まで対応）
4. 「➕ 当日シフトに追加」を押す

### 7-5. MIXトグル

画面下部の「③ 本日の状況一覧」テーブルに「MIX」列がある。

- チェックを入れると、その日のシフトにMIX手当（¥1,500）が適用される
- MIXテーブルに入った日だけチェックを入れること
- 変更は即座にDBに保存される

## 8. 封筒リスト（大会前日〜最終日）

### 8-1. 封筒リストを確認する

1. 「✉️ 封筒リスト」ページを開く
2. イベントを選択
3. 端数処理を選択（推奨：100円単位で切り上げ）

### 8-2. 銀行で用意する現金

ページ上部に**紙幣・硬貨の必要枚数**が表示される。

| 表示 | 意味 |
|------|------|
| 総額 | 全スタッフの支払い合計 |
| 封筒数 | 作成する封筒の総数 |
| 端数切り上げ分 | 端数丸めで増えた金額 |
| 紙幣・硬貨の必要数 | 1万円札○枚、5千円札○枚... |

**この情報を持って銀行に行き、必要な紙幣を準備する。**

### 8-3. 端数処理オプション

| 選択肢 | 効果 | 小銭の量 |
|--------|------|----------|
| なし（そのまま） | 端数あり | 多い |
| 100円単位で切り上げ | 10円・50円がなくなる | 少ない |
| 500円単位で切り上げ | 100円以下がなくなる | かなり少ない |
| 1000円単位で切り上げ | 硬貨がなくなる | 紙幣のみ |

### 8-4. 封筒ラベル

ページ下部で各スタッフの支払明細を確認できる。
展開すると以下が表示される：

- 項目別金額（基本給・深夜・交通費・手当）
- 合計金額
- 紙幣内訳（1万円札×何枚...）

### 8-5. CSV出力

「📥 CSVダウンロード」ボタンで全スタッフの支払い一覧をCSV出力できる。
経理ソフトへの取り込みや帳簿作成に使用。

| 指標 | 説明 |

Dealer・Floor・TD・DC・Chipごとの人数・合計・平均が一覧表示される。

### 9-4. 小口経費

| 項目 | 入力内容 |
|------|----------|
| 日付 | 経費発生日 |
| 内容 | 例: タクシー代（会場→ホテル） |
| 金額 | 円 |
| 申請者 | 誰が使ったか |

### 9-5. 未処理リスト

未払い・領収書未受領のスタッフが一覧表示される。
全員が「支払済み＋領収書受領済み」になるまで、ここをチェックする。

### 9-6. CSV出力

- 支払い一覧CSV: 全スタッフの金額・状態・領収書状況
- 小口経費CSV: 小口経費の全記録

## 11. 承認・支払いフロー

### 11-1. 3段階の状態遷移

```
⏳ 未承認（計算直後）
    ↓ 承認者が「承認」ボタンを押す
✅ 承認済
    ↓ 領収書を受領する
    ↓ 「支払済みにする」ボタンを押す
💴 支払済
```

### 11-2. 各状態でできること

| 状態 | 再計算 | 承認 | 支払い | 領収書 |
|------|--------|------|--------|--------|
| ⏳ 未承認 | ✅ 可能 | ✅ 可能 | ❌ 不可 | ✅ 可能 |
| ✅ 承認済 | ❌ スキップ | — | 領収書受領後のみ | ✅ 可能 |
| 💴 支払済 | ❌ スキップ | — | — | — |

### 11-3. 一括承認

1. 「💰 支払い計算」ページを開く
2. 承認者名を入力（例: 半谷）
3. 「✅ 未承認N名を一括承認」ボタンを押す

### 11-4. 領収書ルール

- **領収書未受領 → 支払い不可**（システムで強制）
- 個別スタッフ詳細で「🧾 領収書受領済み」ボタンを押す
- 領収書受領後に「💴 支払済みにする」が有効になる

### 11-5. 監査ログ

「💰 支払い計算」ページ下部に操作ログ（直近20件）が表示される。

記録される操作：
- 支払い計算の実行
- 承認（承認者名付き）
- 支払い完了
- 領収書受領
- 凍結退勤（人数付き）
- 小口経費の追加

## 12. 計算ルール詳細

### 12-1. 時給計算

| 時間帯 | 計算 |
|--------|------|
| 22:00以前 | 稼働時間 × 通常時給 |
| 22:00以降 | 稼働時間 × 深夜時給 |

深夜時給は通常時給の125%（法定割増率）。
例: 通常¥1,500 → 深夜¥1,875

### 12-2. 休憩控除

| 総稼働時間 | 控除される休憩 |
|-----------|---------------|
| 6時間以下 | なし |
| 6時間超〜8時間以下 | 45分 |
| 8時間超 | 60分 |

休憩時間は通常時間（22時前）から控除される。

### 12-3. 手当

| 手当 | 条件 | 金額 |
|------|------|------|
| 交通費 | 出勤した日 × 日額 | ¥1,000/日（デフォルト） |
| フロア手当 | Floor役職の出勤日 | ¥3,000〜5,000/日 |
| MIX手当 | MIXフラグONの日 | ¥1,500/日 |
| 精勤手当（全日） | イベント全日出勤 | ¥10,000 |
| 精勤手当（4日+） | 4日以上出勤 | ¥6,000 |

### 12-4. 精勤手当の詳細

| イベント日数 | 全日出勤 | 条件付き（¥6,000） |
|------------|---------|-------------------|
| 6日間 | 6日 → ¥10,000 | 4日以上 → ¥6,000 |
| 4日間 | 4日 → ¥10,000 | 4日以上 → ¥6,000 |
| 3日間 | 3日 → ¥10,000 | 2日以上 → ¥6,000 |
| 2日間 | 2日 → ¥10,000 | 1日以上 → ¥6,000 |

### 12-5. 欠勤・遅刻・早退の影響

| 状況 | 給与への影響 | 精勤手当への影響 |
|------|-------------|----------------|
| 欠勤 | その日の給与なし（交通費もなし） | 出勤日数から除外 |
| 遅刻 | 実際の到着時刻から計算（短くなる） | 出勤日数にはカウント |
| 早退 | 実際の退勤時刻まで計算（短くなる） | 出勤日数にはカウント |
| 凍結退勤 | 凍結時刻まで計算（短くなる） | 出勤日数にはカウント |

## 13. よくある操作

### 大会前のチェックリスト

- [ ] イベントを作成した
- [ ] 日別レートを設定した（特に特別日）
- [ ] シフトCSVを取り込んだ
- [ ] 支払い計算を実行した
- [ ] 封筒リストで総額・紙幣枚数を確認した
- [ ] 銀行で必要な紙幣を準備した

### 大会中の毎日の作業（5分）

1. 出退勤ページを開く
2. 日付を選択
3. 「全員出勤」ボタンを押す
4. 来てない人を「欠勤」タブで記録
5. 遅刻・延長・早退・凍結があれば記録
6. MIXテーブルに入ったスタッフのMIXチェックを入れる

### 大会最終日の作業

1. 出退勤ページで「全員退勤」を押す
2. 延長者がいれば記録
3. 支払い計算ページで「再計算」を実行
4. 承認者名を入力して「一括承認」
5. 封筒リストページで最終金額・紙幣内訳を確認
6. スタッフに封筒を渡す → 領収書を受け取る
7. 支払い計算ページで「領収書受領済み」「支払済み」を記録

### 大会後の作業

1. 精算レポートページで未払い・未受領を確認
2. 小口経費を入力
3. CSVを出力して帳簿に反映

## 14. トラブルシューティング

### 支払い額が変わらない

**原因:** 出退勤を記録した後、支払い計算の「再計算」ボタンを押していない。
**対処:** 「💰 支払い計算」ページで「🔄 支払い額を計算」を押す。

### 再計算しても金額が変わらない

**原因:** そのスタッフが「承認済み」または「支払済み」になっている。
**対処:** 承認済み/支払済みのスタッフは上書き保護されている。修正が必要な場合はDBを直接操作する（管理者に依頼）。

### 「全員出勤」を押しても一部が出勤にならない

**原因:** 現在時刻より後に出勤予定のスタッフはスキップされる。
**対処:** 正常動作。そのスタッフの出勤予定時刻になってから再度押すか、個別に出退勤タブで記録する。

### CSVを取り込んだが人数が少ない

**原因:** CSVの列構成がフォーマットと異なる可能性。
**対処:** 「プレビュー」を確認し、スタッフと時刻が正しく読み取れているか確認する。

### 封筒リストに「支払いデータがありません」と表示される

**原因:** 支払い計算がまだ実行されていない。
**対処:** 「💰 支払い計算」ページで計算を実行してから封筒リストを確認する。

### 凍結退勤で対象者がいない

**原因:** 対象スタッフが「出勤中」または「未確定」でない。
**対処:** 先に「全員出勤」を実行してから凍結退勤を行う。

### ブラウザに接続できない

**原因:** Streamlitが起動していない。
**対処:** ターミナルで起動コマンドを再実行する。

## 15. 用語集

| 用語 | 意味 |
|------|------|
| **シフト** | スタッフの勤務予定（日付・開始時刻・終了時刻） |
| **通常時給** | 22:00以前の時間に適用される時給 |
| **深夜時給** | 22:00以降の時間に適用される時給（通常の125%） |
| **フロア手当** | Floor役職のスタッフに支給される日額手当 |
| **MIX手当** | MIXテーブルを担当した日に支給される手当 |
| **精勤手当** | 一定日数以上出勤した場合に支給されるボーナス |
| **凍結（フリーズ）** | トーナメントでテーブルが閉じること。該当ディーラーは退勤 |
| **小口経費** | タクシー代・備品等の少額経費。業務委託費とは別管理 |
| **承認** | 計算結果を半谷氏等が確認・承認すること |
| **pending** | 未承認状態（計算直後） |
| **approved** | 承認済み状態 |
| **paid** | 支払い完了状態 |
| **監査ログ** | 誰が・いつ・何をしたかの操作記録 |
| **端数丸め** | 支払額の端数を切り上げて小銭を減らす処理 |
| **レート** | 日別の時給・手当設定 |
| **CSV** | カンマ区切りテキストファイル。Excel・Google Sheetsから出力可能 |
