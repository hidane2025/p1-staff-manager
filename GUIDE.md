## 1. このツールでできること

| やりたいこと | 画面 | 1人あたり時間目安（※実測ではなく推定） |
|---|---|---|
| 大会の枠組みを作る | 📋 イベント設定 | 5分 |
| スタッフを登録する | 👥 スタッフ管理（Googleフォーム可） | 1人30秒 |
| シフト表を取り込む | 📅 シフト取込（CSV） | 10秒 |
| 給与を自動計算する | 💰 支払い計算 | 全員で30秒 |
| 当日のチェックイン・退勤 | 🕐 出退勤／🎰 ピット端末 | 1人10秒 |
| 封筒を用意する | ✉️ 封筒リスト（紙幣枚数自動） | 全員で1分 |
| 領収書を電子で配る | 📄 領収書発行（PDF＋URL） | 全員で1分 |
| 契約書を電子で結ぶ | ✍️ 契約書発行→📝 電子署名 | 1人2分 |
| 年末に法定調書対象を抽出 | 📆 年間累計（¥50万超を自動フラグ） | 30秒 |

**手計算で5分/人かかっていた支払い作業が、30秒/人に短縮できる**のがこのツールのキモです（出典: 旧 `MANUAL.md` の運用所感）。

## 2. はじめに（ログイン・パスワード）

### ログイン手順
1. 本番URLを開く
2. PII（本名・住所・支払額）を扱う画面を開くと、**ログイン画面が出ます**
3. ユーザーID（例：`nakano`）とパスワードを入力 → 🔓 ログイン

ログインすると左サイドバーに「🔐 <ユーザー名>（admin/viewer） ログイン時刻」が表示され、ログアウトボタンも出ます。

### パスワードの保管
- 1Password / Bitwarden 等のパスワードマネージャを推奨
- **LINE・チャット・メール本文に書かない**
- 90日に1回変更（カレンダーリマインダ）

### ロール（権限）
| ロール | できること |
|---|---|
| **admin** | 全画面の閲覧と編集 |
| **viewer** | 一部画面の閲覧のみ（編集不可） |

### ログイン失敗・パスワード忘れ
- 失敗は監査ログに記録されます（10回以上で要注意扱い）
- パスワードを忘れたら、新しいハッシュを生成して中野さんに Secrets 更新を依頼：
  ```bash
  python scripts/make_app_user.py --username <ユーザーID>
  # --username は必須。実行後にパスワード入力プロンプトが出る。
  ```

## 3. 大会1回の業務フロー

```
大会1ヶ月前  →  大会前週  →  大会当日       →  大会翌日〜1週間内
─────────────────────────────────────────────────
イベント作成    シフト取込    出退勤・打刻       支払い承認・封筒
レート設定      スタッフ登録  ピット端末で確定   領収書発行
                              個別手当          契約書署名追跡
                                              精算レポート
```

ホーム画面の「**今日のTo-Do（8項目）**」を上から順に処理すれば締めまで届きます。
各タスクには ✅（done）／⏳（待機）／⚠️（警告）／❌（未着手）の状態が自動でつきます。

## ① 作る — イベント設定

📋 **イベント設定** ページで大会の「型」を1画面で組み立てます。

### 新規作成
1. ページ上部「新規イベント」セクション
2. 以下を入力：
   - **イベント名**（例：`P1CIRCUITinOSAKA`）
   - **会場**（例：`ヒルトン大阪`）
   - **開始日／終了日**（YYYY-MM-DD）
3. 「作成」を押す

### 日別レート設定
イベント作成後、同じページの「日別レート」セクションで：
| 項目 | 通常日 | 特別日（年末等） |
|---|---|---|
| 通常時給 | 1500 | 1600 |
| 深夜時給（22〜翌5時） | 1875 | 2000 |
| 精勤手当（最終日） | 5000 | 10000 |

### JSONテンプレ取込（おすすめ）
`docs/event_templates/` に過去大会のJSONテンプレあり。同じ会場・期間の大会ならテンプレを1ファイル投入で全レート設定が完了します。

### 地域別交通費ルール（任意）
🚃 **交通費** ページで、東海／**近畿**／関東 ごとの上限を設定（例：東海 ¥3000、近畿 ¥10000）。地域コードは `utils/region.py` の REGIONS で定義（北海道・東北・関東・甲信越・北陸・東海・**近畿**・中国・四国・九州・沖縄）。

## ② 入れる — スタッフ管理／シフト取込

### スタッフ管理（👥）

#### 単発登録
ページ上部「新規登録」フォームで1人ずつ：
- **NO.**（通し番号・任意）
- **ディーラーネーム**（現場での呼び名・必須）
- **本名**
- **住所**
- **メール／電話**
- **役職**（Dealer / Floor / TD / Chip / DC）
- **雇用区分**（業務委託 / タイミー / 正社員）
- **タイミー希望時給**（タイミーのみ）

#### 一括取込（CSV/フォーム）
「📥 スタッフ一括登録/更新」セクションで4つの入口：

| タブ | 用途 |
|---|---|
| 📁 CSVアップロード | 手元で整理した精査済みリストを一括投入 |
| 📋 テキスト貼り付け | スプレッドシートから直接コピペ |
| ✏️ テーブル入力 | 画面上で直接編集 |
| 🔗 Googleフォーム連携 | Google フォームの回答CSVを投入 |

**精査済みスプレッドシートの取込**（中野さんの想定運用）は **「📁 CSVアップロード」** か **「🔗 Googleフォーム連携」** を使います。

#### 名寄せ（2026-06 追加の新機能）
取込時、システムは同一人物を自動で見分けます：

```
照合順： NO. → メール → ディーラーネーム
正規化： 全角半角・空白・大文字小文字をすべて吸収
```

「Eve Kat」「ＥＶＥ　ＫＡＴ」「evekat」は **同じ人** と判定されて、既存IDに統合されます。

**「🔎 名寄せの確認事項 N件」が出たら**、別人取り違えがないか必ず目で確認してください。表記揺れの同一視や同名衝突を「自動でやった」ことを透明化するための表示です。

#### Google フォーム連携の準備
1. Google フォームを `docs/gform_staff_onboarding_template.md` のテンプレに従って作成
2. 回答スプレッドシートを「ファイル → ダウンロード → CSV」で保存
3. 「🔗 Googleフォーム連携」タブで CSV をアップロード
4. プレビューで内容確認 → 「🔄 P1にインポート」

### シフト取込（📅）
1. 対象イベントを選択
2. 「CSV / TSV」セクションでファイルを選択
3. プレビュー → 「取り込み実行」

NO. でスタッフを照合（無ければ自動作成）、日付・予定時刻・MIX フラグを `p1_shifts` に投入します。

## ③ 計算 — ピット端末／支払い計算／封筒リスト

### ピット端末（🎰・タブレット運用推奨）
**退勤打刻＋確定計算を1画面で**完結させるモード。給与窓口は確認だけで済むようになります。

1. NO. を入力（または選択）
2. 実退勤時刻を入力（前日深夜跨ぎは「前日の続き」自動検出）
3. 「確定」を押す → 即座に金額確定

iPad の Cmd+P で印刷専用ビューも出ます。

#### 📦 弁当配布チェック（2026-06-18 追加）
イベント選択直後のエキスパンダー「📦 弁当配布チェック（当日の出勤予定者）」で：
- 対象日を選択 → 出勤予定者一覧が表示（欠勤者は自動除外）
- 各人に「✅ 受領」「🚫 辞退」「🔄 戻す」の3ボタン
- 「✅ 全員受領にする」で一括処理（朝礼後の一斉配布等）
- 上部に **配布対応 X/Y人** の進捗バー

> 列追加（`docs/db_migrations/20260618_add_lunch_status.sql`）の適用が前提。未適用のうちはボタンが反応しません。

### 支払い計算（💰）
1. イベント選択
2. 「全員計算」または「個別計算」
3. 計算結果を確認（pending 状態）
4. 内容OKなら「承認」（pending → approved）

#### 計算ロジックの中身
```
基本給 = 時給 × 勤務時間（深夜帯は深夜時給）
深夜手当 = 0〜5時帯の追加加算
個別手当 = 言語/人材確保/リーダー（オフレコ）
精勤手当 = 全日出勤の場合のみ最終日に加算
交通費   = 地域別上限と領収書から算出
─────────────────────────────────
合計（基本＋深夜＋手当＋精勤＋交通）
端数調整（紙幣最小化のため切り上げ・切り下げ）
確定額（payable_amount）= 封筒・領収書・年間累計で一致
```

#### タイミー
スタッフ登録時の **`custom_hourly_rate`** が優先されます。

#### 凍結退勤
TDが「凍結退勤」マークすると、支払いは強制的に pending に戻り、再計算ルートに乗ります。

### 封筒リスト（✉️・最終日の現金準備）
1. イベント選択
2. 「印刷モード ON」→ Cmd+P で PDF 保存
3. 「銀行で準備する現金」セクションに 1万円札・5千円札・千円札の合計枚数

**封筒に入れる金額は確定額（payable_amount）で、明細にも内訳が印字されます**。スタッフが見ても説明できる構成です。

## ④ 渡す — 個別手当／領収書／契約書

### 個別手当（🎁・オフレコ運用）
言語手当・人材確保手当・リーダー手当を個別付与。
- **金額・理由を個別入力**
- 給与窓口担当だけが見られる（封筒額に統合）
- 計算済みの支払いに対して **加算→確定** の順

> ⚠️ **2026-06-26時点で本番DB未対応**：マイグレ `docs/db_migrations/20260508_add_individual_allowances.sql` を Supabase SQL Editor で実行するまで、画面操作はエラーになります。

### 領収書発行（📄）
1. **発行者設定（🏢）** で Pacific 情報・インボイス番号を1回登録
2. 「📄 領収書発行」で対象支払いを選択 → 「PDF一括生成」
3. 原本／控えの2部、税額内訳の ON/OFF を選択
4. 完成 PDF はそのまま渡すか、**スタッフ向けDL URL**（**7日有効**）を発行
5. スタッフは URL を開くだけ → 自分の領収書だけ DL できる（他人の領収書は見えない）

### 契約書発行（✍️ → 📝 電子署名）
#### 1. テンプレ準備（93）
- 業務委託契約・NDA のテンプレを `pages/93_契約書テンプレ.py` で編集
- docx／pdf／プレーンテキスト取込対応
- 仮版（透かし）→ 正規版（透かしなし）の昇格管理あり

#### 2. 発行（94）
- 対象スタッフを選択 → 「一括発行」
- 各スタッフに **署名URL**（トークン付き・**既定 14日**有効）を配信

#### 3. 署名（99 ・スタッフ操作）
- スタッフが URL を開く
- 同意チェック → 画面に手書きサイン（streamlit-drawable-canvas）→ 送信
- 署名済PDFが Storage に保存され、ハッシュも記録

#### 4. 追跡（94）
- 「署名状況」セクションで「未署名／署名済」を一覧化

#### 二重送信ガード
連打されてもアプリ層（`apply_signature`）が status を事前チェックして拒否（CR-1 / M-4 対応）。※DB側の原子的ガードではなくアプリ層のチェックなので、極端な同時刻アクセスでは取りこぼしの可能性が残る。

## 8. 大会中の操作（出退勤・例外）

### 基本の流れ（1日5分）
🕐 **出退勤** または 🎰 **ピット端末**：
1. 朝：「全員出勤」（来てない人だけ後で個別に「欠勤」マーク）
2. 退勤は予定時刻で **自動確定**（延長/早退だけ修正）
3. 例外（欠勤・遅刻・延長・早退・凍結）を記録

### 例外タブ
| タブ | 用途 |
|---|---|
| 凍結 | TDが凍結マーク → 給与窓口で再計算 |
| 欠勤 | 当日欠勤を即マーク |
| 遅刻／延長／早退 | 実時刻を入力（差分が給与に反映） |
| 個別リセット | 1人だけ pending に戻す |
| 全員リセット | 全員 pending に戻す（大会まるごと再計算時） |

### 当日スタッフ追加
出退勤画面の下部「当日スタッフ追加」で、NO.・氏名・役職を入れて即時登録（後でスタッフ管理画面で本名・住所等を補完）。

### MIX トグル
1卓に複数役職を兼ねる場合のフラグ。封筒明細に「MIX手当」として出ます。

## 9. 締めの作業（精算レポート・年間累計）

### 精算レポート（📊・大会終了直後）
- 支払い状況サマリー（未承認／承認済／支払済 の人数と合計）
- 雇用区分別集計（業務委託／タイミー／正社員）
- 役職別集計（Dealer／Floor／TD／Chip／DC）
- 小口経費入力
- 未処理リスト（受領待ち・承認待ち の漏れ）
- CSV出力（経理・税務向け・受取人確認チェック付）

### 年間累計（📆・確定申告期・法定調書対象抽出）
- 年単位（1/1〜12/31）の全スタッフ累計
- **¥500,000超を自動フラグ**（法定調書提出対象）
- 雇用区分・役職フィルタ
- 「⚠️ 法定調書提出対象者」セクションに 1人ずつ展開 → 本名・住所・メール・支払額を一覧表示
- **本名・住所未登録のスタッフは赤エラー**（補完を促す）
- CSV出力（経理担当・税理士に渡す前提・⚠️ T2個人情報の警告つき）

### 健全性チェック
年間累計の最下部「🩺 スタッフ情報健全性チェック」で、本名・メール・住所のいずれかが未登録のスタッフを抽出。年末調整・法定調書作成前に補完を促します。

## 10. 困ったとき（トラブル対処）

### 「Oh no. Error running app.」が出た
1. https://share.streamlit.io にログイン
2. p1-staff-manager の右端「⋮」→ **Reboot**
3. 1〜2分待って再アクセス
4. それでもダメなら中野さんに連絡（Manage app のログを見て切り分け）

### ログインできない
- ユーザーID か パスワードが違う（監査ログにエラー記録されます）
- パスワードを変えた直後 → Streamlit Cloud で Save 後、アプリの自動再起動を待つ（30秒）

### 領収書PDFが出ない
- 「発行者設定（🏢）」で Pacific 情報を1回登録したか確認
- 「すべて」フィルタを使えば pending でも生成自体は可能だが、**業務的には approved 後の発行を推奨**（フローを乱さないため）

### 契約書のスタッフ署名が反映されない
- 署名URLの有効期限切れ（**既定 14日**）→ 再発行
- スタッフ画面がトークン無しで開かれている → URL コピーミス。再送

### 名寄せで別人扱いされた
- 名寄せは「NO. → メール → 源氏名」の優先順
- 別人ならOK、同一人物なのに別IDになっているなら：
  - 古いNO.のスタッフを 1_スタッフ管理 で開き、メールアドレスを入れる
  - 次回取込で「メール一致で統合」される

### CSVが文字化けする
- Excel で開くと文字化けする場合は、ファイル→開く→「テキストファイルウィザード」で UTF-8 を指定
- ダウンロードCSVは BOM 付き UTF-8 で出しているので、最新Excelなら通常開けます

### 一括取込で行が落ちる／エラーが出る
- **CSV / TSV 一括取込**（`bulk_import_staff`）は `name_jp`（ディーラーネーム）が空の行のみエラー化（その他フィールドは寛容）
- **Googleフォーム連携** は `real_name` 空・メール形式不正・電話桁数不足を「⚠️ バリデーションエラー N件（続行は可能）」として一覧表示
- いずれもインポート画面の「⚠️ エラー N件」「⚠️ バリデーションエラー N件」をクリックすると詳細展開

## 11. 安全運用ルール

### やってはいけないこと
- スタッフのマイナンバーをこのツールに登録しない（オフライン管理）
- 銀行口座情報をこのツールに登録しない（払込先別管理）
- パスワード本文をチャット・LINE・メール本文に書かない
- 退職したスタッフに招待リスト（Streamlit Cloud Sharing）を残さない

### やるべきこと（月次・5分）
- [ ] 監査ログの異常チェック（深夜帯の閲覧・10回以上のログイン失敗）
- [ ] 月初の手動バックアップ実行
- [ ] Streamlit Cloud Sharing の招待リスト確認（退職者掃除）
- [ ] `pip list --outdated` で脆弱性アップデート確認

### やるべきこと（四半期・30分）
- [ ] ADMIN_PASSWORD の更新（90日サイクル）
- [ ] Storage バケット全件ローカル保管
- [ ] anon 直接アクセスのRLS抜き打ちテスト

### やるべきこと（年次・1月・60分）

## 12. よくある質問

### Q1: スマホで使えますか？
A: 使えますが、印刷ビュー・封筒リストなど画面の広いページは **iPad / PC 推奨** です。出退勤打刻ぐらいなら iPhone でもOK。

### Q2: 同時に複数人が触っても大丈夫？
A: 大丈夫です（Streamlit セッションは別々で動きます）。ただし、同じスタッフの支払いを2人同時に編集すると後勝ちになるので、役割分担推奨。

### Q3: 過去の大会データを削除したい
A: 通常は残します（年間累計・法定調書のため）。削除は SQL 直接実行が必要なので、中野さんに依頼してください。

### Q4: 「タイミー」のスタッフを別枠で計算したい
A: スタッフ登録時に「雇用区分＝タイミー」を選び、`custom_hourly_rate` を入れれば自動で別計算されます。タイミーは支払調書対象外なので年間累計の集計でも区分が分かれます。

### Q5: 契約書テンプレを差し替えるとき、過去の契約は再生成される？
A: されません。**本文スナップショット**が契約書ごとに保存されているので、過去契約は当時の本文のまま参照可能（改ざん防止のため）。

### Q6: スタッフが「自分の領収書をなくした」と言ってきた
A: 91_領収書発行 で対象を再選択 → 「URLを再発行」で新しい**7日有効**のDL URLを送ります。原本（控え）は **PDF を ZIP 化** して一括ダウンロードできます。

### Q7: 「精勤手当」を一部スタッフだけ付けたくない
A: 当日「凍結退勤」マークか「欠勤」マークを入れると、自動的に精勤手当が剥奪されます。明示的な減額は個別手当画面でマイナス金額を入れる方法もあります。

### Q8: 法定調書って何ですか？
A: 報酬を年間50万円超支払った場合に税務署へ提出する書類。源泉徴収の有無に関わらず提出義務があります。このツールは対象者の自動抽出と必要情報（本名・住所・支払額）のCSV出力までやります。実際の作成・提出は経理・税理士に渡してください。

### Q9: 源泉徴収はこのツールでやりますか？
A: いいえ、**税額計算は経理担当が担当**します。このツールは「人別の年間支払額を正確に集計する」ところまでが責任範囲です（2026-06 で名寄せを強化したのはこの正確性のため）。

### Q10: Reboot ボタンを押したら、進行中のデータは消えますか？
A: **消えません**。データはすべて Supabase（クラウドDB）に保存されています。Reboot はアプリのプロセス再起動だけで、データには触りません。ただし、編集中で未保存のフォーム入力は消えます。
